## 二、代码审查发现

### 2.1 代码质量问题

#### 问题1: Rate Limiter 清理逻辑非线程安全 ⚠️

**位置**: `backend/app/utils/rate_limiter.py` 第42-70行

**问题描述**: `_cleanup_old_requests` 方法在多线程/异步环境下可能有竞态条件

```python
def _cleanup_old_requests(self):
    """清理过期的请求记录"""
    current_time = time.time()
    
    if current_time - self.last_cleanup < self.cleanup_interval:
        return
    
    # 遍历清理 - 无锁保护
    for ip in list(self.ip_requests.keys()):
        ...
```

**影响**: 在高并发场景下，可能导致缓存字典操作冲突

**优先级**: P2

### 2.2 潜在 Bug/边界情况

#### 问题4: WebSocket 重连未验证 Token 有效性 ⚠️

**位置**: `api/v2/websocket.py`

**问题描述**: WebSocket 自动重连时未检查 Token 是否已过期，可能导致使用过期 Token 尝试连接

**当前代码**:
```python
async def connect(self, websocket: WebSocket, token: str):
    # 验证 token
    payload = self.auth_manager.validate_token(token)
    # ... 连接建立
    
    # 重连时直接使用原 token，未检查过期时间
```

**影响**: 长时间运行的服务可能使用过期 Token，导致认证失败

**优先级**: P2

---

#### 问题5: 缓存键冲突风险 ⚠️

**位置**: `services/cache.py`

**问题描述**: 如果 Redis key 包含特殊字符，可能导致缓存数据覆盖错误

**当前实现**: 直接使用传入的 key，未做规范化处理

**优先级**: P3

---

#### 问题6: 交易服务异步任务未处理取消 ⚠️

**位置**: `services/trading_service.py` 第170行

**问题描述**: 
```python
asyncio.create_task(
    webhook_manager.send_webhook(...)
)
```

**问题**: 创建异步任务后未保存引用，无法取消，可能导致资源泄漏

**优先级**: P2
