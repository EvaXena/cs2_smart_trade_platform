## 三、发现的问题

### 3.1 P2 问题：RuntimeWarning 警告未消除

**位置**: 
- `app/core/risk_manager.py:302`
- `app/core/risk_manager.py:810`
- `app/services/trading_service.py:383`

**问题描述**:
测试运行时仍出现 RuntimeWarning，表示有些协程未被正确 await：

```
RuntimeWarning: coroutine 'AsyncMockMixin._execute_mock_call' was never awaited
```

**根因分析**:
- pytest-asyncio 的 mock 配置问题
- 在某些异步调用路径上缺少 await
- 测试全部通过，但警告影响测试输出可读性

**建议修复**:
1. 检查 risk_manager.py 第302行和810行的异步调用链
2. 在 trading_service.py 第383行添加正确的 await
3. 或在测试中配置 `pytest.warns` 忽略特定警告

---

### 3.2 P2 问题：配置硬编码

**位置**: 多处

| 文件 | 位置 | 硬编码值 | 建议 |
|------|------|----------|------|
| `api/v1/endpoints/monitoring.py` | 26 | `error_rate_threshold = 10.0` | 移入配置 |

| `services/trading_service.py` | 51 | `DEFAULT_TIMEOUT = 30` | 移入配置 |

**影响**: 部署时需要修改代码才能调整阈值

**建议**: 统一配置管理，支持运行时配置更新

---

### 3.3 P3 问题：缓存键未规范化

**位置**: `services/cache.py`

**问题描述**: 如果 Redis key 包含特殊字符，可能导致缓存数据覆盖错误

**当前实现**: 直接使用传入的 key，未做规范化处理

**建议**: 添加 key 前缀和校验

---

### 3.4 P3 问题：WebSocket 重连未验证 Token

**位置**: `api/v2/websocket.py`

**问题描述**: WebSocket 自动重连时未检查 Token 是否已过期

**影响**: 长时间运行的服务可能使用过期 Token，导致认证失败

**建议**: 在重连前检查 Token 过期时间
