## 三、可改进功能点

### 3.1 P2 改进点：消除 DeprecationWarning

**问题描述**: 
测试运行时出现11个 DeprecationWarning，影响测试输出可读性。

**位置**:
1. `app/services/cache.py:1513` - get_cache() 在初始化前被调用
2. `app/api/v1/endpoints/monitoring.py:221, 258, 343` - 缓存初始化问题

**建议修复**:
```python
# 当前代码
cache = get_cache()

# 建议修改为
from app.services.cache import ensure_cache_initialized
await ensure_cache_initialized()
cache = get_cache()
```

### 3.2 P2 改进点：配置硬编码

**位置**: 多处

| 文件 | 位置 | 硬编码值 | 建议 |
|------|------|----------|------|
| `api/v1/endpoints/monitoring.py` | 26 | `error_rate_threshold = 10.0` | 移入配置 |
| `api/v1/endpoints/monitoring.py` | 28 | `response_time_threshold = 2000.0` | 移入配置 |
| `services/trading_service.py` | 51 | `DEFAULT_TIMEOUT = 30` | 移入配置 |

**影响**: 部署时需要修改代码才能调整阈值

### 3.3 P3 改进点：缓存键未规范化

**位置**: `services/cache.py`

**问题描述**: 如果 Redis key 包含特殊字符，可能导致缓存数据覆盖错误

**建议**: 添加 key 前缀和校验机制

### 3.4 P3 改进点：WebSocket 重连未验证 Token

**位置**: `api/v2/websocket.py`

**问题描述**: WebSocket 自动重连时未检查 Token 是否已过期

**建议**: 在重连前检查 Token 过期时间
